# Тест-кейсы

## TC-001: Создание клиента с валидными, сгенерированными данными

**Название**: Проверка успешного создания клиента (*Add Customer*) с использованием сгенерированных данных по заданным правилам.

### Предусловия
- Пользователь находится на странице, с которой можно инициировать добавление клиента.

### Шаги

#### 1. (Подготовительный шаг) Сгенерировать тестовые данные:
- **a.** Сгенерировать **Post Code** из 10 цифр.  
  *Пример данных*: `0001252667`
- **b.** Сгенерировать **First Name** на основе Post Code `0001252667`:
  - i. Разбиваем: `00`, `01`, `25`, `26`, `67`
  - ii. Конвертируем (по правилу `N % 26 = индекс буквы`):
    - `00 % 26 = 0` → `'a'`
    - `01 % 26 = 1` → `'b'`
    - `25 % 26 = 25` → `'z'`
    - `26 % 26 = 0` → `'a'`
    - `67 % 26 = 15` → `'p'`  
  *Пример данных (First Name)*: `abzap`
- **c.** Подготовить остальные обязательные поля.  
  *Пример данных (Last Name)*: `Testov`

#### 2. (Шаг выполнения) Начать добавление клиента:
- **a.** Нажать кнопку **"Add Customer"**.

#### 3. (Шаг выполнения) Заполнить форму:
- **a.** Ввести конвертируемые данные в поле **"First Name"**.
- **b.** Ввести `Testov` в поле **"Last Name"**.
- **c.** Ввести cгенерированныe 10 цифр в поле **"Post Code"**.

#### 4. (Шаг выполнения) Сохранить клиента:
- **a.** Нажать кнопку **"Add Customer"** (или **"Save"**) для сохранения формы.

### Ожидаемый результат
- Пользователь видит модальное окно об успешном создании клиента.
- Форма добавления клиента очищается.
- При переходе в общий список клиентов (на вкладку **"Customers"**), в таблице присутствует новая запись с данными.

## TC-002: Сортировка клиентов по имени (First Name)

**Название**: Проверка корректной работы сортировки списка клиентов по столбцу **"First Name"** (по возрастанию и убыванию).

### Предусловия
- Пользователь находится на странице списка клиентов (**"Customers"**).
- В системе существует как минимум 3 клиента с разными именами, например:
  - Клиент 1: `"Neville"`
  - Клиент 2: `"Albus"`
  - Клиент 3: `"Voldemort"`
- Изначально список не отсортирован по **"First Name"**.

### Шаги

#### 1. Один раз нажать на заголовок столбца **"First Name"**.

##### Ожидаемый результат (1)
- Список клиентов отсортирован по алфавиту (**A–Z**).
- Порядок отображения клиентов: `"Albus"`, `"Neville"`, `"Voldemort"`.
- Рядом с заголовком **"First Name"** отображается индикатор сортировки (например, стрелка вверх ▲).

#### 2. Нажать на заголовок столбца **"First Name"** еще раз.

##### Ожидаемый результат (2)
- Список клиентов отсортирован в обратном алфавитном порядке (**Z–A**).
- Порядок отображения клиентов: `"Voldemort"`, `"Neville"`, `"Albus"`.
- Индикатор сортировки меняет направление (например, стрелка вниз ▼).

## TC-003: Удаление клиента с длиной имени, близкой к средней

**Название**: Проверка удаления клиента, длина имени которого наиболее близка к среднему арифметическому длин всех имен.

### Предусловия
- Пользователь находится на странице списка клиентов (**"Customers"**).
- В системе существует ровно 3 клиента с данными:
  - Клиент 1: `"Albus"`
  - Клиент 2: `"Neville"`
  - Клиент 3: `"Voldemort"`

### Шаги

#### 1. (Подготовительный шаг) Определить клиента для удаления:
- **a.** Список имен: `["Albus", "Neville", "Voldemort"]`
- **b.** Длины имен: `[5, 7, 9]`
- **c.** Среднее арифметическое длин: `(5 + 7 + 9) / 3 = 7`
- **d.** Имя, чья длина (`7`) ближе всего к среднему (`7`) — это `"Neville"`

#### 2. (Шаг выполнения) Найти и удалить целевого клиента:
- **a.** В списке клиентов найти строку, где **"First Name"** = `"Neville"`.
- **b.** Нажать кнопку **"Delete"** для этой строки.

#### 3. (Шаг выполнения, если применимо) Подтвердить удаление:
- **a.** Если система показывает диалоговое окно (например, *"Are you sure you want to delete Neville?"*), нажать **"Yes"** или **"Confirm"**.

### Ожидаемый результат
- Пользователь видит сообщение об успешном удалении клиента.
- Клиент `"Neville"` исчезает из списка клиентов.
- Клиенты `"Albus"` и `"Voldemort"` по-прежнему отображаются в списке.
